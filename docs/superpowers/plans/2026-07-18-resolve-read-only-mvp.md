# Resolve Read-Only MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build an installable Zendesk private app that gives agents a ticket-aware Claude chat with cited, read-only WooCommerce, ShipStation v1/v2, and Zendesk-history skills.

**Architecture:** A React/ZAF v2 ticket-sidebar app sends secure requests through Zendesk's proxy to a Cloudflare Worker. The Worker runs a bounded Anthropic tool loop, executes commerce tools, delegates Zendesk tools back to ZAF, and persists ticket-scoped conversations in D1 for 90 days. Versioned skill packs depend on shared contracts and a small skill SDK, so new reviewed skills can be added without changing the chat UI or orchestration core.

**Tech Stack:** TypeScript, pnpm workspaces, React, Vite, ZAF v2, Zod 4, Anthropic TypeScript SDK, Cloudflare Workers, D1, Wrangler 4+, Vitest 4+, Cloudflare's Vitest integration, Testing Library, MSW, ESLint, and Prettier.

## Global Constraints

- The first release is read-only. Do not implement Zendesk profile writes or ShipStation mutations in this plan.
- Product name is `Resolve`.
- Use the Quiet Console visual language in a wide, flexible Zendesk ticket side panel.
- Use Anthropic Claude behind a provider interface.
- Support one Zendesk account/private installation.
- Support WooCommerce REST API v3.
- Support ShipStation API v2 and legacy API v1 behind one normalized adapter.
- Search Zendesk live under the current agent's permissions; do not build a copied ticket index.
- Persist ticket-scoped chat for 90 days in D1.
- Do not persist credentials, authorization headers, raw provider payloads, or hidden model reasoning.
- Secure values are Zendesk installation settings used only as header placeholders in `secure: true` requests to the allow-listed Worker.
- Complete responses are non-streaming in the MVP.
- Every tool input and provider response is schema-validated.
- Unknown object keys are rejected with Zod `z.strictObject`.
- A write-risk tool must declare `requiresConfirmation: true`; the registry rejects any violation even though no write handlers ship in this milestone.
- New skills are reviewed source modules, not runtime code uploads.
- Use a current Cloudflare compatibility date (`2026-07-18` when this plan is executed) and enable `nodejs_compat`.
- Generate Worker binding types with `wrangler types`; do not hand-write an `Env` interface.
- Use structured logs and timing-safe backend-token comparison.
- Do not create commits unless the user explicitly authorizes commits during execution.

- [ ] **Step 4: Implement the contracts and registry**

`types.ts` defines a strict reviewed-code contract:

```typescript
import type { ZodType } from "zod";

export type ToolRisk = "read" | "write";
export type ToolExecution = "server" | "delegated";

export interface ToolExecutionContext {
  signal: AbortSignal;
  credentials: Readonly<Record<string, string | undefined>>;
  tenantKey: string;
  ticketId: number;
}

export interface SkillCredential {
  settingName: string;
  headerName: string;
  required: boolean;
  secret: boolean;
}

export interface ToolDefinition<TInput, TOutput> {
  name: string;
  description: string;
  risk: ToolRisk;
  requiresConfirmation: boolean;
  execution: ToolExecution;
  inputSchema: ZodType<TInput>;
  outputSchema: ZodType<TOutput>;
  handler?: (input: TInput, context: ToolExecutionContext) => Promise<TOutput>;
}

export interface RegisteredTool {
  name: string;
  description: string;
  risk: ToolRisk;
  requiresConfirmation: boolean;
  execution: ToolExecution;
  inputSchema: ZodType;
  outputSchema: ZodType;
  handler?: (input: unknown, context: ToolExecutionContext) => Promise<unknown>;
}

export interface SkillDefinition {
  id: string;
  name: string;
  version: string;
  instructions: string;
  credentials: readonly SkillCredential[];
  tools: readonly RegisteredTool[];
  healthCheck?: (
    context: ToolExecutionContext,
  ) => Promise<{ ok: boolean; message: string }>;
}

export function defineTool<TInput, TOutput>(
  tool: ToolDefinition<TInput, TOutput>,
): RegisteredTool {
  if (tool.execution === "server" && !tool.handler) {
    throw new Error(`Server tool ${tool.name} requires a handler`);
  }
  if (tool.execution === "delegated" && tool.handler) {
    throw new Error(`Delegated tool ${tool.name} cannot define a handler`);
  }
  const typedHandler = tool.handler;
  const base: Omit<RegisteredTool, "handler"> = {
    name: tool.name,
    description: tool.description,
    risk: tool.risk,
    requiresConfirmation: tool.requiresConfirmation,
    execution: tool.execution,
    inputSchema: tool.inputSchema,
    outputSchema: tool.outputSchema,
  };
  if (!typedHandler) return base;
  return {
    ...base,
    async handler(input, context) {
      const parsedInput = tool.inputSchema.parse(input);
      const output = await typedHandler(parsedInput, context);
      return tool.outputSchema.parse(output);
    },
  };
}
```

```typescript
import { z } from "zod";
import type {
  RegisteredTool,
  SkillDefinition,
  ToolExecutionContext,
} from "./types";

const Identifier = z.string().regex(/^[a-z][a-z0-9_]*$/);

export class SkillRegistry {
  readonly skills: readonly SkillDefinition[];
  private readonly tools = new Map<
    string,
    { skill: SkillDefinition; tool: RegisteredTool }
  >();

  constructor(skills: readonly SkillDefinition[]) {
    const skillIds = new Set<string>();
    for (const skill of skills) {
      Identifier.parse(skill.id);
      if (skillIds.has(skill.id))
        throw new Error(`Duplicate skill id ${skill.id}`);
      skillIds.add(skill.id);

      for (const tool of skill.tools) {
        Identifier.parse(tool.name);
        if (tool.risk === "write" && !tool.requiresConfirmation) {
          throw new Error(`Write tool ${tool.name} must require confirmation`);
        }
        if (this.tools.has(tool.name)) {
          throw new Error(`Duplicate tool name ${tool.name}`);
        }
        this.tools.set(tool.name, {
          skill,
          tool,
        });
      }
    }
    this.skills = skills;
  }

  getTool(name: string) {
    return this.tools.get(name);
  }

  listTools() {
    return [...this.tools.values()];
  }

  async executeServerTool(
    name: string,
    input: unknown,
    context: ToolExecutionContext,
  ): Promise<unknown> {
    const registered = this.tools.get(name);
    if (!registered) throw new Error(`Unknown tool ${name}`);
    const { tool } = registered;
    if (tool.execution !== "server" || !tool.handler) {
      throw new Error(`Tool ${name} is delegated`);
    }
    return tool.handler(input, context);
  }
}
```

`auth.ts` hashes both values to fixed-length buffers before comparison:

```typescript
const encoder = new TextEncoder();

async function digest(value: string): Promise<ArrayBuffer> {
  return crypto.subtle.digest("SHA-256", encoder.encode(value));
}

export async function authenticate(
  request: Request,
  expectedToken: string,
): Promise<boolean> {
  const header = request.headers.get("authorization");
  const supplied = header?.startsWith("Bearer ") ? header.slice(7) : "";
  const [actualHash, expectedHash] = await Promise.all([
    digest(supplied),
    digest(expectedToken),
  ]);
  return crypto.subtle.timingSafeEqual(actualHash, expectedHash);
}
```
